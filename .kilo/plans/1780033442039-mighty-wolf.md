# Plan: Dashboard Routes, Auth Modal, Intro Fixes, and Click Handling

## Issues Identified

### 1. Auth Modal Centering Issue
**Problem:** Sign-in popup appears off-center and non-functional on desktop
- `AuthModalHost.tsx:45-57` uses `onMouseDown` on backdrop for closing, but buttons use `onClick` - potential race condition
- `SignInCard.tsx` uses `GlowCard` + `GradientGlowFrame` which have complex pointer-event handling (`pointer-events-none` on children)
- The modal animation with framer-motion might delay click responsiveness

### 2. Intro Sequence Bug
**Problem:** Sequence shows: home page â†’ intro â†’ home page (incorrect), should be: home page â†’ intro â†’ stay on home page
- `SiteIntroGate.tsx:74-75` - children render immediately while overlay shows on top
- No proper loading state to wait for Spline assets before showing content
- Intro overlay might not have the correct z-index to block interactions during the animation

### 3. Intro Expiry
**Problem:** Currently marks intro as permanently seen; user wants hourly expiry or show on hard refresh
- `intro-storage.ts` uses boolean flag `"1"` for permanent storage
- Need to change to timestamp-based check with hourly expiration

### 4. Intro Text Size
**Problem:** Text in intro is too large (`text-4xl sm:text-5xl md:text-6xl`)
- Needs reduction for better visual balance

### 5. Double-Click Required Everywhere
**Problem:** Buttons require multiple clicks to register
- `AuthModalHost.tsx:45` - `onMouseDown` on backdrop may intercept first click
- `GlowCard.tsx:160-168` - wrapper uses `pointer-events-none` styles that may interfere
- `AnimatedGroup.tsx` - framer-motion wrappers add DOM layers that can delay interaction
- Event propagation issues between backdrop and content

## Implementation Plan

### Phase 1: Fix Auth Modal Centering & Click Handling

1. **`frontend/components/auth/AuthModalHost.tsx`**
   - Change `onMouseDown` to `onClick` on backdrop for consistent event handling
   - Ensure proper z-index stacking (backdrop: z-50, content: z-60)
   - Add `pointer-events-auto` to ensure content is interactive

2. **`frontend/components/ui/sign-in-card.tsx`**
   - Simplify modal structure - remove complex `GlowCard` for auth modal
   - Ensure `w-full max-w-md` properly constrains width on all viewports
   - Add `pointer-events-auto` to the card container

3. **`frontend/components/ui/glow-card.tsx`**
   - Review `pointer-events-none` on inner elements - may cause click issues
   - Ensure interactive elements within the card can receive clicks

### Phase 2: Fix Intro Sequence

1. **`frontend/components/landing/SiteIntroGate.tsx`**
   - Add loading state to wait for Spline preload before rendering children
   - Ensure proper z-index hierarchy (children behind, overlay on top)
   - Use `position: fixed` for intro to guarantee it covers content
   - Only hide overlay after exit animation completes

2. **`frontend/lib/intro-storage.ts`**
   - Change storage key format to include timestamp
   - Add function to check hourly expiration
   - `hasSeenIntro()` checks if last shown hour differs from current hour
